# Agent 全生命周期纪律（Core — 零平台依赖）

5 条规则覆盖 Define → Plan → Build → Verify → Review → Ship → RUN → HEAL → EVOLVE 全部 9 阶段。

任何 AI Agent 项目，引入本文件即可获得完整的编码+运行+进化纪律。

---

## 规则 1：声明假设（Surface Assumptions）

**阶段：** Define → Build

**为什么：** Agent 闷头写，跑完发现全错——因为假设从来没说出来，没人有机会纠正。

**怎么做：** 任何非平凡任务开始前，先写：

```
我的假设：
1. [API/数据/环境的假设]
2. [输入格式的假设]
3. [边界条件的假设]
→ 如果不对，告诉我再开始。
```

**豁免：** 改 3 行、修已知 bug、跑已验证脚本 → 不需要。新功能、改核心逻辑 → 必须。

---

## 规则 2：范围自律（Maintain Scope）

**阶段：** Build → Ship

**为什么：** 修一个 bug，顺手重构了无关模块 → 下游全崩。

```
✅ 可以碰的：任务指定的文件、任务指定的函数
❌ 不能碰的：
  - 「顺手」重构无关模块
  - 「顺便」改注释/格式/变量名
  - 「发现」一个不相关的问题就去修
  - 删除你不完全理解的代码
```

**唯一例外：** 发现的问题会导致崩溃或数据丢失 → 可以修，但必须**单独 commit**。

**检查方法：** `git diff --stat` 看改了哪些文件，超范围 → 回退。

---

## 规则 3：运行守护（RUN）

**阶段：** Ship → RUN

**为什么：** 脚本在后台跑，崩了没人知道，直到数据歪了才发现。

```
✅ 必须做的：
  - 每个定时任务有心跳文件（更新最后存活时间）
  - 每个脚本写退出码和错误日志
  - API 配额追踪 + 90% 预警 + 100% 熔断
  - {N} 分钟无心跳 → 自动报警（不杀进程，先诊断）

❌ 不能做的：
  - 挂了不报错（exit 0 掩盖一切）
  - 配额耗尽还继续重试
  - 静默跳过异常（except: pass）
```

**平台适配：** 心跳检查工具 = `{PLATFORM}_health_check`，通知渠道 = `{PLATFORM}_notify`。

---

## 规则 4：自愈优先诊断（HEAL）

**阶段：** RUN → HEAL

**为什么：** 盲目杀进程可能把限流窗口越撞越小。先诊断再动作。

```
自愈链（不可跳步）：
  1. 诊断 → 读日志/健康文件，确认根因
  2. 匹配 → 90% 失败是已知模式，走固定修复规则
  3. 修复 → 让进程自己恢复（如切凭证），不替代进程做事
  4. 验证 → 修复后等一个周期，确认恢复再汇报
  5. 杀进程 → 仅当前 4 步无效时使用

❌ 禁止：
  - 不看日志就杀进程
  - 用 LLM 分析已知模式的失败
  - 修完不验证就报"已修复"
```

**平台适配：** 已知失败模式 = `{PLATFORM}_known_patterns`，健康文件 = `{PLATFORM}_health_dir`。

---

## 规则 5：经验回写（EVOLVE）

**阶段：** HEAL → EVOLVE

**为什么：** 经验停在聊天记录里 = 下次还会踩同一个坑。

```
必须回写：
  ✅ 发现新失败模式 → 更新已知模式库
  ✅ 同一坑踩 {N} 次 → 固化到代码（不只是文档）
  ✅ 修复 tricky bug → 追加到对应 Skill 的 Pitfalls
  ✅ 发现更好做法 → patch 到对应 Skill

不用回写：
  ❌ 一次性配置问题
  ❌ 外部服务挂了恢复
  ❌ 改了不到 3 行
```

**平台适配：** 已知模式库 = `{PLATFORM}_known_patterns`，Skill 目录 = `{PLATFORM}_skills_dir`。

## 加载方式

Agent 接到任何任务后自动加载本 Skill。第一条消息就声明假设。
