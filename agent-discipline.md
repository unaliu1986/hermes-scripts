# Agent 行为纪律

写任何代码前必读。2 条核心规则，违反一条返修概率翻倍。

---

## 规则 1：声明假设（Surface Assumptions）

**为什么：** Agent 闷头写 2 小时，跑完发现全错——因为假设从来没说出来过，没人有机会纠正。

**怎么做：** 任何非平凡任务开始前，先写：

```
我的假设：
1. [API/数据/环境的假设]
2. [输入格式的假设]
3. [边界条件的假设]
→ 如果不对，告诉我再开始。
```

**真实案例：**
- reverse_price.py：假设万邦 API 可用 → 挂了返回空 → 全标 NO_SUPPLY（本应是 API_FAIL）
- batch_update_v4.py：假设 eBay Token 有效 → 过期 → 4000 条 FAIL

**如果加了声明假设：** Agent 会先说「假设万邦 API 此时可用」，你或 overseer 能提前发现不对。

**小任务豁免：** 改 3 行、修一个已知 bug、跑一个已验证脚本 → 不需要。新脚本、新功能、改核心逻辑 → 必须。

---

## 规则 2：范围自律（Maintain Scope）

**为什么：** Agent 修一个 bug，顺手重构了竞品数据格式 → 下游全崩。

**怎么做：**

```
✅ 可以碰的：任务指定的文件、任务指定的函数
❌ 不能碰的：
  - 「顺手」重构无关模块
  - 「顺便」改注释/格式/变量名
  - 「发现」一个不相关的问题就去修
  - 删除你不完全理解的代码
```

**唯一例外：** 如果你发现的问题会导致脚本崩溃或数据丢失，可以修，但必须**单独 commit**，不能混在这次改动的 commit 里。

**检查方法：** `git diff --stat` 看改了哪些文件，超过任务范围的 → 回退。

## 运行纪律（RUN + HEAL + EVOLVE）

写代码的纪律管到 Ship。代码跑起来之后，还有三个阶段。

---

## 规则 3：运行守护（RUN）

**为什么：** 脚本在 cron 里跑，崩了没人知道，直到数据歪了你才发现。

```
✅ 必须做的：
  - 每个 cron 脚本必须有心跳文件（更新最后存活时间）
  - 每个脚本必须写退出码和错误日志
  - API 配额必须追踪 + 90% 预警 + 100% 熔断
  - 30 分钟无心跳 → 自动报警（不杀进程，先诊断）

❌ 不能做的：
  - 脚本挂了不报错（exit 0 掩盖一切）
  - 配额耗尽还继续重试（浪费调用）
  - 静默跳过异常（except: pass）
```

**检查方法：** overseer_agent.py N1-N4 检查，红灯 → 微信通知。

---

## 规则 4：自愈优先诊断（HEAL）

**为什么：** 盲目杀进程重启可能把限流窗口越撞越小。先诊断再动作。

```
自愈铁律：
  1. 诊断 → 读日志/健康文件，确认根因
  2. 匹配 → 90% 失败是已知模式，走固定修复规则
  3. 修复 → 写控制文件（如 reset_creds），让进程自己切凭证
  4. 验证 → 修复后等一个周期，确认恢复再汇报
  5. 杀进程 → 只有前 4 步无效才走这步

❌ 禁止：
  - 不看日志就杀进程
  - 用 AI 分析已知模式的失败（LLM 调用 = 浪费 Token）
  - 修完不验证就报"已修复"
```

**检查方法：** overseer 红灯 → 匹配 known_patterns → 单 Agent 诊断（不搞三 Agent 会诊）。

---

## 规则 5：经验回写（EVOLVE）

**为什么：** 经验停在聊天记录里 = 下次还会踩同一个坑。

```
什么时候必须回写：
  ✅ 发现新的失败模式 → 更新 heal_submit_fails.py 的 known_patterns
  ✅ 同一个坑踩了 3 次 → 固化到代码里（不只是 SKILL 里加文字）
  ✅ 修复了一个 tricky bug → 追加到对应 SKILL 的 Pitfalls 段
  ✅ 发现更好的做法 → patch 到对应 SKILL

什么时候不用回写：
  ❌ 一次性配置问题
  ❌ 外部服务挂了恢复（不是你的系统问题）
  ❌ 改了不到 3 行
```

**检查方法：** 每月一次 skill-stocktake，检查哪些 Skill 该更新了。

| 你想干什么 | 看哪个 |
|---|---|
| 这活能不能不做？ | Ponytail 六阶梯 |
| 代码写完了，质量过关吗？ | code-review-gate（11 道闸门） |
| 跑完了，结果是真的吗？ | anti-rationalization-gate（8 条防自欺） |
| 出 bug 了怎么修？ | 铁律：先确认根因 → 不擅改 → 小样再全量 |
| 新脚本要跑 7x24？ | script-boundary-card（4 维边界卡） |
| 多 Agent 协作？ | 各自只能写自己的目录，通过健康文件交接 |

## 加载方式

Agent 接到任务后自动加载本 Skill。第一条消息就声明假设。

违反者：返修概率 ~70%（从你过去半年的生产事故统计）。
